HashCode

Dec 30th, 2011

Java中的每一个对象都拥有hashCode()方法来生成该对象的hash code，那么这个hash code是做什么用的呢？

首先要说起hash table，结构如下：

hash code 就是对象在数组中的索引的位置，当然也有情况是几个对象的hash code 值一样（如2 ,5 ），这种情况叫做 collision，我们用链表来存储。

我们再返回到这个问题上，例如, 你新建了一个hash map来存放一个叫做Rect类以及对Rect的描述： Map map = HashMap(); map.put(new Rect(4,5), "rect"); 同时又通过get方法来获取描述，如: String description = map.get(new Rect(4,5))

在这种情况下，我们所获取得到的是null值。你也许会想，“哦，因为是new出来的对象，所以两个肯定不等，需要override object.equals() 方法”。然而，override之后，依然取到的是null值。

因为在JAVA中，hash table的put和get方法中，都有这样的一段判断代码来决定是否在一个位置上存储和读取对象： if ((e.hash == hash) && e.key.equals(key)) 既检查hash code值是否相等，还调用对象的equals方法，因此我们在覆盖equals方法时，同时一定要覆盖hashCode方法来保证相同的object有相同的hash code，如果不这样做的话，当这个class和hash set、hash map、hash table一起工作时将会出错。

因此，关于hashcode()有个通用的约定： A. Whenever it is invoked on the same object more than once during an execution of a Java application, the hashCode method must consistently return the same integer, provided no information used in equals comparisons on the object is modified. This integer need not remain consistent from one execution of an application to another execution of the same application.

B. If two objects are equal according to the equals(Object) method, then calling the hashCode method on each of the two objects must produce the same integer result.

C. It is not required that if two objects are unequal according to the equals(java.lang.Object)) method, then calling the hashCode method on each of the two objects must produce distinct integer results. However, the programmer should be aware that producing distinct integer results for unequal objects may improve the performance of hashtables.

当然了，如果相同的实例能拥有唯一的hash code值，对hash table的性能提升有很大的帮助，固然很好，但是往往是蛮困难的。 那么如何写一个有效的hashCode方法呢？ 在effective java中叙述的非常详细，在这里举个例子: @Override public int hashCode() { int result = 17; result = 31 * result + length; result = 31 * result + width; return result; } 如果class的content是String或其他实例的话，可以考虑使用.hashCode()来计算result值。

In-memory Database

Nov 4th, 2011

Definition

An in-memory database is a database that runs entirely in main memory, without touching a disk. Often they run as an embedded database: created when a process starts, running embedded within that process, and is destroyed when the process finishes.

(From Martin Fowler , InMemoryTestDatabase)

Contrast

Much faster than disk-optimized database, because simpler algorithms,fewer CPU instructions, fewer I/O reading activity. Lack support for durability portion of ACID, because it will lose all information when lose power or reset. But they can add durability via using some mechanisms, such as snapshot files,transaction logging,Non-volatile random, access memory, high availability.

When to use it ?

Using in-memory database in a enterprise application can dramatically reduce build time Not touching real database in unit test. it is safe for real database, keep database independent and clean. Don’t worry about the sequence of test caused different db result .

Structure of Function Test in Our Project

Nov 3rd, 2011

Our project is a python web application based on Django framework, we use selenium as web driver to run our function test. Directory structure of function test is like this: –func_tests —-framework —-drivers —-utils —-pages —-indexpage —-projectpage … … —-resources —-testdata —-tests —-indexpagetests —-projectpagetests *******… …

Framework directory : In drivers directory: we wrap original web driver into costumed driver which offers more functions about choosing webdriver and selecting web elements . In utils directory: we offer basic operate functions for dealing with database, meanwhile, wrap basic web element to different specify elements such as button_web_element, drop_down_web_element.

pages directory: Create a single page object for each page you want to test, page object offers basic element locator 、navigation、response message and so on .

Resources directory: Put some local settings files for function test here, which are same as setting files in APP.

Testdata directory: Basic testdata for more than 1 function test , such as account info, initial date for database and so on.

Tests directory: All function tests are here, each page test is a single directory, also, end to end tests are here. In each test directory , there is page test file and corresponding test data.

Be Careful Auto Import in Your IDE

Nov 2nd, 2011

In our project , we have two kinds of database, postgresql and couchdb. For postgresql, when we want to add a new model related table in django, we should extend models.Model class and add some filed like this:

class ProgresTableModel(models.Model):
 message = CharField(max_length=160)
 organization = ForeignKey(Organization)
 status = BooleanField(default=False)
 ......

Here, we use BooleanField from django.db.models.fields, import like this:

from django.db.models.fields import BooleanField

For couchdb, we should extend Document class , and like this :

class CouchdbModel(Document):
 document_type = TextField()
 status = BooleanField()
 ......

Here, we use BooleanField from couchdb.mapping import like this:

from couchdb.mapping import  BooleanField

It looks well, unfortunately we used both in 1 python file, and the pycharm auto imported the first one, when we create database it doesn’t throw out any exception , BooleanField object in different package is totally different.

So, be careful auto import function in your IDE !

An Interesting Command in MacOS

Nov 1st, 2011

Today i just knew a funny command in mac os, it is command say.

This command can convert the input text to audible speech, for example :

say hello world

you can hear a male voice to read it .Also you can let it read a file using “-f file_name”, also you can record the voice to the file using “-o output.aiff ” .

The most interesting is that using “-v voice” to use different voice, for example:

say -v good hello world # high pitch

say -v bad hello world #low pitch
